## 一、气质与意象

### 当前状态
- **印象**：功能完整的紫微斗数计算工具 ✅
- **感觉**：暗黑 + 金色点缀，风格正确但缺乏情感冲击
- **缺失**：没有建立起"神秘东方仪式感"的情绪体验

### 优化方向

**核心意象**：「古卷启封 · 星尘入境」

> 想象一位身着墨色长袍的隐士，在烛光中缓缓展开一幅泛黄的丝绢古卷。卷上星图随着指尖触碰而泛起金色微光——不是数据展示，而是一场宇宙祝祷的仪式。

**气质关键词**：
- 神秘 · 静谧 · 仪式感 · 东方奢华
- 不卖弄科技感，而是"古老智慧借现代载体呈现"

**意象锚点**：
| 概念 | 视觉投射 |
|------|----------|
| 命盘 | 星图丝绢卷轴，水波纹扩散 |
| 十二宫 | 古代宫殿飞檐 + 星光环绕 |
| 主星 | 颤动发光的舍利/星辰 |
| 四化 | 金木水火四种元素的韵律波动 |
| 起盘 | 盖印仪式——朱砂落章，命盘显现 |

## 二、存在的关键设计问题

### 问题 1：品牌名称与核心产品错位 ⚠️

| 维度 | 现状 | 问题 |
|------|------|------|
| 品牌名 | `Mystic Zodiac` | "Zodiac"暗示西方星座/生肖 |
| 核心产品 | 紫微斗数 | 纯正东方星命术 |
| 用户预期 | 星座/生肖相关 | 实际是紫微命盘 |

**建议**：在品牌调性中强化"东方神秘学"，弱化"Zodiac"的误导。当前阶段可暂时保留名称，但副标题应明确为 `紫微斗数 · 东方命理`。

### 问题 2：仪式感与当前体验严重脱节 🔴

**设计文档描绘的旅程**（理想）：
```
启动 → 星尘动画 → 痛点共鸣(3问) → 生辰录入(3步) → 
起名印鉴 → 命盘揭幕(动画) → 沉浸浏览
```

**代码实现的旅程**（现实）：
```
启动(3.5s倒计时) → 表单输入 → 图表展示
```

> 缺少了最核心的**情感爬坡**——用户没有产生"将要揭晓天机"的期待感。

### 问题 3：视觉质感缺乏"东方奢华感"

| 元素 | 现状 | 问题 |
|------|------|------|
| 星曜符号 | Emoji (👑🐺☀️) | 表情符号缺乏精致感 |
| 字体 | 未加载的 Cinzel | Fallback 到 system font |
| 交互 | 无动画过渡 | 页面切换生硬 |
| 宫位网格 | 基础 CSS grid | 缺乏丝绢卷轴质感 |
| 四化标识 | 小色块 + 文字 | 可做元素律动特效 |

### 问题 4：功能架构散落，未形成产品闭环

```
已开发的模块：
✅ 紫微算法引擎（核心资产）
✅ 命盘展示（基础可用）
✅ 每日运势（随机算法）
✅ 周易占卜（6/64卦）
✅ 性格引擎（内容丰富）
✅ 分享卡片生成器（完整）
✅ 支付模块（框架）
✅ 合规模块（框架）
✅ AI 配置（DeepSeek）

但：以上模块彼此割裂，没有形成统一的产品体验
```

## 三、优化方案（分阶段实施）

### 🔵 Phase 1：视觉升级（可立即执行）

#### 1.1 建立字体系统
```css
/* 务必在 HTML 中预加载 */
@import url('https://fonts.googleapis.com/css2?family=Cinzel:wght@400;500;600;700;900&family=Inter:wght@300;400;500;600&family=Noto+Serif+SC:wght@400;600;700&display=swap');

/* 应用规则 */
--font-title: 'Cinzel', serif;         /* 英文标题 / 装饰 */
--font-body: 'Inter', sans-serif;       /* 英文正文 */
--font-classical: 'Noto Serif SC', serif; /* 中文古典文本 */
```

#### 1.2 替换 Emoji 为自绘矢量符号
- 为 14 主星、6 辅星、12 生肖分支各设计 SVG 符号
- 过渡方案：使用 Unicode 符号替代 Emoji（✦ ✧ ◈ ◆ ⚘ ⟡ 等）
- 最终目标：使用 inline SVG 组件，可随主题变色

#### 1.3 加入页面过渡动画
- 页面切换：`fadeIn + scale(0.98→1)` 200ms ease-out
- 命盘揭幕：四角光芒汇聚 → 中心向外扩散
- 宫位点击弹窗：`scale(0.9→1) + opacity(0→1)` 带 spring 弹性

#### 1.4 色彩系统的精准调校

```css
/* 当前颜色（保留） */
--color-bg-primary: #0A0A0F;
--color-bg-card: #14141F;
--color-gold: #C9A96E;
--color-purple: #8B5CF6;

/* 新增微调 */
--color-gold-glow: rgba(201, 169, 110, 0.15);  /* 金色辉光背景 */
--color-border-subtle: rgba(201, 169, 110, 0.08); /* 极细金线 */
--color-ink-wash: rgba(232, 228, 223, 0.03);   /* 墨染纹理 */
--gradient-cosmic: linear-gradient(180deg, #0A0A0F 0%, #0F0F1A 40%, #14141F 100%);
```

### 🟡 Phase 2：体验闭环（核心优化）

#### 2.1 重建 Onboarding 流程
按照"三幕式叙事"结构重构：

**第一幕：共鸣（痛点引入）**
```
你最近是否感到：
◈ 人生方向模糊，不知下一步该往何处？
◈ 感情关系中出现重复的冲突模式？
◈ 事业遇到瓶颈，不知该守还是该进？

（用户勾选后，系统记录痛点用于后续解读）
```

**第二幕：仪式（生辰录入 → 起名印鉴）**

将当前扁平的表单改造成三步向导：
1. 选择出生年月日（日期滚轮/日历）
2. 选择出生时辰（时辰选择器，带生肖动物提示）
3. 输入姓名 → 生成"命盘印鉴"（包含姓名+生辰+五行局的朱砂印章风格图片）

**第三幕：揭幕（命盘展现）**

用户点击"揭晓命盘"后：
1. 屏幕暗下，出现墨滴扩散动画
2. 金色光线从中心向外勾勒十二宫格
3. 主星逐个降落到对应宫位（带有颤动微光效果）
4. 出现命宫主星的注解引言

#### 2.2 命盘解读体验增强
- 点击宫位弹窗时，增加"古卷展开"动画
- 弹窗内加入五行动态色块
- 主星带发光效果（该宫的 SiHua 星星尤其明显）
- 支持左右滑动切换宫位（当前是点一个关一个）

#### 2.3 Oracle 增加 Ziwei 算法权重
- 当前 Oracle 是随机分数，应改为基于命盘四化的动态计算
- 连接当前日的天干地支与命盘的交互
- 引入"流日、流月"基本概念

---

### 🟢 Phase 3：功能整合（中后期）

#### 3.1 上线 AI 解读
- DeepSeek 配置已就绪，需要完整的 Prompt Engineering
- 将痛点选择结果 + 命盘数据 + 四化 拼接成结构化 prompt
- AI 输出流式展示（打字机效果）

#### 3.2 连接分享卡片到 UI
- Chart 页添加"分享命盘"按钮 → 生成 Destiny Card
- Oracle 页添加"分享今日天机"按钮 → 生成 Daily Whisper Card
- 使用 Canvas 2D 渲染（已实现）

#### 3.3 统一算法数据源
- 将 `app/src/utils/ziwei.ts` 统一引用 `ziwei/` 包的导出
- 避免两处代码维护

## 五、设计禁忌

| 禁止事项 | 原因 |
|----------|------|
| ❌ 使用 Emoji 作为星曜符号 | 不够精致，破坏沉浸感 |
| ❌ 纯黑阴影（`#000000`） | 在有配色背景上显脏 |
| ❌ 科技蓝/青色渐变 | 偏离东方神秘基调 |
| ❌ 直角/无圆角的容器 | 失去"古卷"应有的柔润 |
| ❌ 表格风格的分割线列表 | 应使用卡片容器替代 |
| ❌ 无加载状态的点击 | 仪式感需要"过程" |
| ❌ 中文与英文混排不设定不同字体 | 中西文混排需要分别控制 |
| ❌ 强制用户一次性输入所有信息 | 三步向导比大表单体验更好 |
